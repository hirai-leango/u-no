# u-no.me メディア記事フォーマット v1

キラキラテキーラの構成をベースに、u-no.me 向けにSEO施策と構造化データを組み込んだ**記事の型**。AI量産時はこの骨子＋各記事のKWを渡す。

---

## 0. 記事の全体像（上から順に）

```
パンくず（BreadcrumbList構造化）
H1：{主KW}＋ベネフィット＋「完全ガイド／入門」
リード（120〜300字）：定義先出し → 一般イメージとのギャップ → 記事の価値
📌 この記事でわかること（3項目・箇条書き）
✅ この記事のポイント（3〜5項目・箇条書き）
目次（H2/H3から自動生成・アンカーリンク）
─────────────────────────
本文：H2 ×5〜7（各H2にKW/検索意図、必要ならH3で分解）
  └ 各セクションに「表・例文・手順」を最低1つ（＝差別化）
─────────────────────────
よくある質問（FAQ 4〜6問／各100〜150字）→ FAQPage構造化
まとめ（主KW再掲＋次の一歩）
CTA（固定文 → u-no.me）
関連記事（内部リンク3〜5本）
```

**想定文字数：** ピラー 3,000〜4,000字／通常 2,000〜2,500字（キラキラは5〜6千字だが、u-no.meは"読ませて即行動"寄りなので中量で密度重視）

---

## 1. 見出し設計ルール（意図的SEO）

| 要素 | ルール | 例 |
|---|---|---|
| **H1** | 主KWを**完全一致**で含める＋末尾にベネフィット語 | 他己紹介とは？ビジネスで信頼を伝える新しい自己紹介 |
| **H2** | 「{KW}？{答え/ベネフィット}」形式。"定義"だけの無味な見出しにしない | 他己紹介と自己紹介の違い——信頼はどちらで生まれるか |
| **H3** | 具体で分解（例文・手順・比較・ケース） | ビジネスでの使いどころ：転職／営業／フリーランス |
| **共起語** | 本文に検索意図の周辺語（信頼・第三者・推薦・実名 等）を自然に散らす | — |
| **サジェスト** | 「とは／例文／書き方／違い／頼み方」等の複合語をH2/FAQに配置 | — |

> ポイント：**見出しだけ読めば記事の要旨が分かる**こと（スキャナビリティ＝滞在時間＝SEO）。

---

## 2. 目次（TOC）の実装意図

キラキラには明示的な目次が無いが、**u-no.meでは入れる**（UX＋SEO）。

- **効果**：①ページ内ジャンプでUX向上・滞在/回遊増　②Googleがジャンプリンク（サイトリンク）を認識しSERPに表示され得る　③記事の網羅性が視覚的に伝わる
- **実装**：`@nuxt/content` の `toc` を使い、H2・H3を自動抽出してアンカー付きで冒頭に表示。各見出しに `id` を付与。

## 4. よくある質問（FAQ）

- **4〜6問**、各回答**100〜150字**。本文で答えた要点を、検索クエリそのままの疑問文で再提示。
- 質問文は**サジェスト/PAA（他の人はこちらも質問）由来の実クエリ**を使う。
- ⚠️ **重要（正直な注意）**：Googleは2023年以降、**FAQリッチリザルト（★付き表示）を公的機関・医療系サイトに限定**。一般サイトでは星は基本つかない。
  → それでも **FAQは入れる価値大**：①強調スニペット／PAA枠の獲得　②UX・網羅性　③構造化データ自体は将来の再開に備えて無害。**"星狙い"ではなく"クエリ網羅"目的**と割り切る。

## 5. 内部リンク・CTA

- **本文中ディープリンク 3〜5本**：関連記事へ文脈内リンク（アンカーテキストにKW）。
- **末尾に関連記事 3〜5本**（同クラスター＋ピラー）。
- **CTA固定文**（本文中1＋末尾1）：
  > 自己申告では伝わらない信頼を、他己紹介で。**u-no.me で無料のプロフィールをつくる →**

## 6. 構造化データ（JSON-LD）テンプレート

記事ページの `<head>` に1つの `@graph` で出力。`{}` を各記事の値で置換。

```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@graph": [
    {
      "@type": "BreadcrumbList",
      "itemListElement": [
        { "@type": "ListItem", "position": 1, "name": "ホーム", "item": "https://u-no.me/" },
        { "@type": "ListItem", "position": 2, "name": "メディア", "item": "https://u-no.me/media/" },
        { "@type": "ListItem", "position": 3, "name": "{カテゴリ名}", "item": "https://u-no.me/media/{category}/" },
        { "@type": "ListItem", "position": 4, "name": "{記事タイトル}" }
      ]
    },
    {
      "@type": "Article",
      "headline": "{記事タイトル（H1・全角32字前後まで）}",
      "description": "{メタディスクリプション（120字前後）}",
      "image": "https://u-no.me/media/{slug}/ogp.png",
      "datePublished": "{2026-07-15T09:00:00+09:00}",
      "dateModified": "{2026-07-15T09:00:00+09:00}",
      "author": { "@type": "Organization", "name": "ユーノーミー編集部", "url": "https://u-no.me/" },
      "publisher": {
        "@type": "Organization",
        "name": "ユーノーミー（u-no.me）",
        "logo": { "@type": "ImageObject", "url": "https://u-no.me/favicon-192.png" }
      },
      "mainEntityOfPage": { "@type": "WebPage", "@id": "https://u-no.me/media/{slug}/" }
    },
    {
      "@type": "FAQPage",
      "mainEntity": [
        { "@type": "Question", "name": "{質問1}", "acceptedAnswer": { "@type": "Answer", "text": "{回答1（100〜150字）}" } },
        { "@type": "Question", "name": "{質問2}", "acceptedAnswer": { "@type": "Answer", "text": "{回答2}" } }
      ]
    }
  ]
}
</script>
```

**注意点**
- `Article` は `datePublished` / `dateModified` / `author` / `publisher.logo` が揃うと評価が安定。更新のたび `dateModified` を更新。
- `BreadcrumbList` は**SERPにパンくず表示される（今も有効）**ので必ず入れる。
- `FAQPage` は上記の通り星は付きにくいが記載はしておく（本文のFAQと**内容一致**必須。ズレるとスパム判定）。
- OGP（`og:image` 等）は別途 `<head>` に。記事用OG画像は動的生成の仕組みを流用可。

## 7. Nuxt実装メモ（@nuxt/content 前提）

各記事Markdownのfrontmatterに構造化データの元データを持たせ、記事レイアウトでJSON-LDを組み立てる。

```yaml
---
title: "他己紹介とは？ビジネスで信頼を伝える新しい自己紹介"
description: "他己紹介とは、第三者があなたの人柄や実績を紹介すること。ビジネスでの意味・例文・使い方を、信頼が伝わる理由とともに解説します。"
category: "他己紹介"
categorySlug: "takoshokai"
slug: "takoshokai-toha"
keywords: ["他己紹介 とは", "他己紹介 ビジネス", "他己紹介 例文"]
publishedAt: "2026-07-15"
updatedAt: "2026-07-15"
ogImage: "/media/takoshokai-toha/ogp.png"
summaryPoints:            # ✅ この記事のポイント
  - "他己紹介は第三者があなたを紹介すること"
  - "自己申告より信頼されやすい"
  - "u-no.meなら他己紹介を常時集められる"
faq:
  - q: "他己紹介とは何ですか？"
    a: "他己紹介とは、自分ではなく第三者があなたの人柄や実績を紹介することです。…"
  - q: "自己紹介との違いは？"
    a: "自己紹介は主観的な自己申告、他己紹介は第三者の客観的な証言です。…"
related: ["takoshokai-reibun", "reference-check-tanomaretara", "meishi-gawari-profile"]
---
```

- 記事レイアウト（`pages/media/[category]/[slug].vue` 等）で frontmatter → JSON-LD を生成
- `<ContentDoc>` ＋ `toc` で本文・目次を描画
- パンくず・関連記事・CTAはレイアウト側で共通化

## 8. 記入例（C1「他己紹介とは」の骨子に当てはめ）

```
H1  他己紹介とは？ビジネスで信頼を伝える新しい自己紹介
リード  他己紹介とは、第三者があなたの人柄や実績を紹介すること。研修のアイスブレイクの印象が強いが、実はビジネスの信頼づくりに効く…
📌 わかること：意味／ビジネスでの使い方／集め方
✅ ポイント：第三者の証言は盛れない／自己申告より信頼される／URLで共有できる
目次（自動）
H2 他己紹介とは？意味と読み方
H2 なぜ今ビジネスで「他己紹介」なのか（自己申告の限界）
  H3 リファレンスチェックが当たり前になった時代
H2 他己紹介・自己紹介・推薦状の違い（比較表）
H2 ビジネスでの使いどころ（転職・営業・フリーランス）
H2 他己紹介を集める方法 → u-no.me（CTA）
よくある質問（4問）
まとめ（主KW再掲＋「まずプロフィールを作る」）
関連記事：他己紹介の例文／リファレンスを頼まれたら／名刺代わりプロフィール
```
